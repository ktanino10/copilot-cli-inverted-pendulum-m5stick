# IPS Command — Live Demo (GitHub Pages)

This folder is the **statically-hosted demo** of the IPS Command UI.

It is automatically generated by `tools/build_demo.py` from the live
Flask template at `tools/templates/index.html`. Every time you change
the template, rerun:

```bash
python3 tools/build_demo.py
```

## How the static demo works

There is no Flask backend on GitHub Pages, so the page injects a
`window.IPS_DEMO = true` flag and a tiny JavaScript shim (already
present in `tools/templates/index.html`) intercepts every `/api/*`
fetch:

| Endpoint                  | Demo behaviour                                         |
| ------------------------- | ------------------------------------------------------ |
| `GET  /api/s`             | returns a simulated device-state object (50 ms updates) |
| `GET  /api/c?q=k=v`       | mutates the simulated state so sliders feel "live"     |
| `GET  /api/sessions`      | reads bundled `data/sessions/index.json`               |
| `GET  /api/sessions/<id>` | reads bundled `data/sessions/<id>.json`                |
| `POST /api/sessions`      | acknowledged but not persisted                         |
| `DELETE /api/sessions/x`  | acknowledged but not persisted                         |

A purple `DEMO MODE` badge appears in the bottom-left corner so
visitors immediately know they are looking at a simulation.

## Enabling GitHub Pages

In the GitHub repo settings:

1. **Settings → Pages**
2. **Source:** *Deploy from a branch*
3. **Branch:** `main`, **folder:** `/docs`
4. Save. After ~1 minute, the demo will be live at
   `https://<owner>.github.io/inverted-pendulum-m5stick/`
   (which redirects to `…/demo/`).

The `.nojekyll` file in `docs/` disables Jekyll so all files (including
the generated `data/sessions/*.json`) are served as-is.
